## このサイトがカバーする保護の範囲

<CardGrid>
  <Card title="保存データ（Data at Rest）" icon="seti:lock">
    ディスク上のデータファイル・バックアップを暗号化し、媒体盗難・流出時のリスクを低減します。

    → **TDE**
  </Card>
  <Card title="転送データ（Data in Transit）" icon="right-arrow">
    クライアント〜DB間の通信を暗号化し、ネットワーク上の盗聴・改ざんを防ぎます。

    → **NNE / TLS（TCPS）**
  </Card>
  <Card title="利用時データ（Data in Use）" icon="list-format">
    クエリ結果を動的にマスキング・フィルタリングし、権限のないユーザーへの情報漏えいを防ぎます。

    → **Data Redaction / VPD / OLS**
  </Card>
  <Card title="認証・認可" icon="approve-check-circle">
    誰がDBにアクセスできるか・何を操作できるかを制御し、なりすましや不正操作を防ぎます。

    → **IAM / MFA / DB Vault / SQL Firewall**
  </Card>
</CardGrid>

## 暗号化・マスキング

ディスク上のデータと通信経路を保護し、漏えい・盗聴リスクを低減します。

<LinkCard
  title="透過的データ暗号化（TDE）"
  description="データファイルを暗号化し、ディスク上のデータ漏えいリスクを下げます。クエリに対しては透過的に復号されるため、利用側の改修を最小化できます。"
  href="./encryption/tde/"
/>
<LinkCard
  title="通信暗号化"
  description="クライアントとDB間の通信を暗号化し、盗聴リスクを低減します。NNE（Oracle Net）と TLS（TCPS）の2方式を扱います。"
  href="./encryption/network/"
/>
<LinkCard
  title="Data Redaction"
  description="クエリ結果に対して動的にマスキングを行い、機微情報の表示を抑止します。データ自体を書き換えず、参照時の見せ方だけを制御できる点が特徴です。"
  href="./masking/data-redaction/"
/>

## アクセス制御

DBへのアクセスや操作を制限し、不正利用・内部脅威を抑止します。

<LinkCard
  title="Virtual Private Database"

  href="./access-control/vpd/"
/>
<LinkCard
  title="Database Vault"
  description="職務分離のための制御を追加し、強力な権限ユーザーの操作も制限できます。保護領域や操作ルールを定義し、運用上の不正や誤操作のリスクを下げます。"

<LinkCard
